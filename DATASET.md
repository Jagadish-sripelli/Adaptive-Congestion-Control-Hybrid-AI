# Dataset Description

This document describes the datasets used in the research project:

**"A Hybrid Bio-Inspired and Machine Learning Framework for Adaptive Congestion Control and QoS Optimization in Modern Networks"**

The datasets are used for training and evaluating the congestion prediction model and routing optimization framework.

---

# Dataset Sources

The dataset used in this study was generated from three primary sources:

1. **Synthetic Traffic Dataset**
   - Generated using NS-3 network simulations
   - Simulates dynamic traffic patterns in heterogeneous networks
   - Used for training the CatBoost congestion prediction model

2. **CAIDA Network Traffic Traces**
   - Real-world internet traffic traces
   - Used to evaluate the framework under realistic conditions
   - Source: https://www.caida.org

3. **MAWI Working Group Traffic Traces**
   - Backbone network traffic data
   - Used to validate system performance in bursty traffic environments
   - Source: https://mawi.wide.ad.jp

---

# Dataset Generation

The synthetic dataset was generated using the **NS-3 network simulator**.

Simulation parameters:

| Parameter | Value |
|----------|-------|
| Number of Nodes | 100–200 |
| Network Topologies | Grid, Random, Scale-Free |
| Mobility Model | Random Waypoint |
| Maximum Node Speed | 5 m/s |
| Traffic Type | CBR and Poisson |
| Packet Size | 256–1024 bytes |
| Simulation Duration | 300 seconds |

Traffic metrics were collected at each node during simulation.

---

# Dataset Features

The dataset includes the following features used for congestion prediction:

| Feature | Description |
|-------|-------------|
| queue_length | Current queue size of the node buffer |
| hop_count | Number of hops in the routing path |
| packet_drop_ratio | Percentage of packets dropped |
| round_trip_time | Time taken for packet round trip |
| bandwidth_utilization | Ratio of used bandwidth to total bandwidth |
| inter_arrival_time | Time between packet arrivals |
| congestion_label | Binary label (0 = normal, 1 = congested) |

# Data Preprocessing

Before training the CatBoost model, the following preprocessing steps were applied:

- Removal of incomplete records
- Normalization of traffic metrics
- Label generation based on queue occupancy threshold (70%)
- Dataset split into training and testing sets (75% / 25%)

---

# Dataset Usage

The dataset is used in two stages of the framework:

### 1. Machine Learning Prediction Layer
The dataset is used to train the **CatBoost classifier** to predict network congestion.

### 2. Hybrid Decision Engine
Predicted congestion states influence routing decisions in the GA–ACO optimization module.
